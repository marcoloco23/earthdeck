# Plan: `narrate` — streamed text notes on the dashboard

**Date**: 2026-06-12
**Status**: COMPLETED
**Phase**: Cross-cutting (user request, Session 7) — richer dashboard storytelling

## Goal

A zero-key `narrate` tool that lets Claude push **rich text notes** (interpretations,
context, warnings) to the live dashboard alongside data cards — and **stream** a growing
narrative by updating the same note across calls.

## Background

User: "add a feature to tool calls that allow you to stream text updates to the dashboard
to provide richer information alongside and interpretations of the data." Today every
dashboard card is data-shaped (imagery/series/…); Claude's interpretation lives only in
the chat. The dashboard pipeline (push → /ingest → SSE → feed) only ever appends new
cards, and the web feed de-dupes by id (SSE replays state on reconnect), so updates are
currently impossible.

## Approach

Streaming model: **upsert by card id**. Each `narrate` call sends the FULL note text;
passing the same `noteId` replaces the existing card in place (server upsert + feed node
replacement). Claude holds the running text and re-sends it grown — no server-side append
state, idempotent on SSE replay, trivially safe.

- New card type `note` (server allow-list + web renderer + styles).
- `payload.text` validated at the trust boundary (1–20 000 chars).
- Renderer: markdown-lite via DOM nodes only (`## ` headings, `- ` bullets, `**bold**`,
  blank-line paragraphs) — textContent everywhere, no innerHTML for the body.
- `kind: info | insight | warning` → accent styling (allow-listed in the renderer too).
- Feed update semantics: replace node **in place** (no jump to top), skip identical
  payloads (replay), don't re-focus the map on update; map focus on first appearance only
  (note has optional bbox).

## Risks / edge cases

- SSE replay after reconnect re-sends every card → identical-payload skip keeps the feed
  stable; updates must not re-trigger map focus.
- Feed bounding (60 nodes) must clean the id→node map on eviction.
- Hostile note text → textContent-only rendering; type/kind allow-listed; length capped.
