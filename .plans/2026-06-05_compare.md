**Date**: 2026-06-05
**Status**: COMPLETED (live-verified, NDVI -0.146)
**Phase**: ROADMAP Phase 4

## Goal

`eo_compare(bbox, dateA, dateB, index="NDVI")`: render the same place at two dates and
compute the index delta (e.g. mean-NDVI drop → deforestation / flood / burn). The headline
"global impact" feature.

## Background

Reuses `CopernicusClient` (Phase 3). Per date: one `process()` render (least-cloudy
composite in a window) + one `statistics()` index. Delta = B − A on mean/median.

The dashboard card needs TWO images (before/after), but today a card stores one image.
Extend the card model to carry multiple images.

## Approach

- Extend the model: `IngestPayload.images?: {mimeType,dataBase64}[]` and `Card.imageUrls?: string[]`.
  Server stores each at `/img/{id}::{n}`, validates each (mime allow-list), broadcasts
  `imageUrls`. Keep the single-`image` path for the other tools unchanged.
- `eo_compare` in `tools/analysis.ts`: 2 renders + 2 index stats; return both images + a
  delta text block; push a `compare` card with `images:[before,after]` and the delta.
- Dashboard: `compare` card renders before/after side-by-side + the Δ; map overlays the
  "after" image (reuse showImagery path) so the location is visible.
